## 概要
- 課題: trackball-typing-practice のUI/UX・ゲーム導線・視認性を段階的に改善
- 目的: 開始前後の操作しやすさとゲーム状態の理解しやすさを高める
- 完了条件: 指定されたUI/挙動修正（オーバーレイ、文言統一、LIFE/BOMB表示、一時停止導線、視認性改善）が反映され、ブラウザで確認できる

## 進行ログ（バトンタッチ順）
1. Planner
- 入力: ユーザーからの複数回の改善要望（開始オーバーレイ、操作導線、HUD、文言、配色、視認性）
- 実施内容: 修正方針を「最小差分で段階実装 + 都度ブラウザ検証」に整理
- 成果物: 実装優先順（操作阻害解消 -> 視認性 -> 文言統一 -> デザイン）
- 次役割への引き継ぎ: UI配置と導線の再設計

2. UI Designer
- 入力: 既存のゲーム画面構成と開始/終了オーバーレイ
- 実施内容: 下部ステータス集約、開始/終了時の背面視認、北欧系ブルー配色方針を定義
- 成果物: LIFE/BOMB中央配置、危険帯の明確化、色トーン統一方針
- 次役割への引き継ぎ: フロント実装へ

3. Frontend Engineer
- 入力: UI方針とユーザー指定要件
- 実施内容:
  - 開始オーバーレイ外の背面操作と視認性を改善
  - Homeボタン削除、文言「モード選択へ」統一
  - ESCで一時停止/再開（3カウント）
  - Bomb化（ゲージ -> ストック運用）
  - LIFEをハート表示化、中央下へ再配置
  - 音量UIと「タイプ開始」UIを削除
  - 敵ロック時ヒント表示をアイコン下固定・INPUT文言削除
  - ロックリングの色制御（ホバー黄色/ロック赤）と重なり調整
  - 危険帯を明確な赤帯へ変更、全体を北欧系ブルーへ統一
  - 敵上部HPバーを削除
- 成果物: `games/trackball-typing-practice.html`, `css/trackball-typing-practice.css`, `scripts/trackball-typing-practice.js`, `scripts/site-layout.js` の更新
- 次役割への引き継ぎ: 文言・可読性確認

4. Copywriter
- 入力: ゲーム内文言、HUD文言、ボタン文言
- 実施内容: 表記ゆれ調整（Topへ戻る -> モード選択へ など）
- 成果物: 一時停止・遷移系文言の統一
- 次役割への引き継ぎ: 実装レビュー

5. Reviewer
- 入力: 実装差分と画面確認結果
- 実施内容: 回帰観点（開始/終了オーバーレイ、操作不能化、下部レイアウト干渉、文言不一致）を重点確認
- 指摘（重大/中/軽微）:
  - 重大: なし
  - 中: 一時的にリング視認性の不安定があり、リング要素方式で修正
  - 軽微: 下部UI重なりを余白調整で解消
- 非回帰チェック結果（既存合意事項が維持されているか）: 維持
- 次役割への引き継ぎ: SEO/計測の影響確認

6. SEO & Analytics Specialist
- 入力: 画面UI変更中心の差分
- 実施内容: メタ情報・計測導線への直接影響なしを確認
- 成果物: 追加のSEO修正なし
- 次役割への引き継ぎ: 最終テスト

7. Performance & Accessibility Tester
- 入力: 最終実装画面
- 実施内容: ブラウザで操作検証（開始/停止/再開/Top遷移、LIFE/BOMB表示、敵ロック表示）
- テスト結果: エラーなし、主要挙動は要件どおり
- 最終判定: 合格
